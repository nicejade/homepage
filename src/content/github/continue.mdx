## 项目介绍

Continue 是面向开发者的开源 AI 编程助手，可作为 VS Code、JetBrains 等环境中的日常插件，也可通过 CLI 承载团队自己的模型、提示词和上下文策略。官方仓库目前已只读维护，但 2.0 版本扩展与文档仍可作为学习与自托管参考。

## 核心特性

- 支持 VS Code、JetBrains 等主流 IDE 插件形态
- 可配置多种模型提供方与上下文来源
- 提供 CLI 与团队级编码代理配置能力
- 开源可审计，便于自定义规则与工作流
- 文档覆盖安装、配置与模型接入全流程

## 对用户价值

Continue 的价值在于可组合：开发者可以选择模型、配置上下文来源，并把重复性的编码、解释和重构任务放进熟悉的 IDE 流程里。对于关注隐私和可控性的团队，它比完全黑盒的托管工具更容易评估和治理。

## 与替代方案

- 相比 GitHub Copilot 等托管助手，Continue 更强调开源、可自托管模型与团队自定义规则。
- 相比纯终端 AI 工具（Aider），Continue 更贴近 IDE 内的日常编码节奏。
- 若需要强自主代理与 Kanban 编排，可评估 Cline 等方案。

## 适应人群

- 希望在 IDE 内使用 AI 编程助手的个人开发者。
- 需要统一团队编码代理配置的工程团队。
- 想把本地模型或私有模型接入开发流程的 AI 工具探索者。

## 如何使用

### 前置条件

- 已安装 VS Code 或 JetBrains 等受支持的 IDE。
- 准备好所用模型的 API 密钥或本地模型端点。

### 安装方式

从 [continue.dev](https://continue.dev) 或编辑器扩展市场安装 Continue 插件。

### 首次运行

打开 IDE 中的 Continue 面板，选择模型提供方并完成密钥配置，再指定项目规则与上下文来源。

### 验证是否成功

在项目中发起一次简单问答或代码补全，确认模型响应正常且上下文文件可被读取。

### 常见坑 / 注意事项

- 官方仓库已只读，新功能与长期支持需关注文档与社区 fork 动态。
- 团队推广前先用小仓库验证模型成本、权限边界与上下文范围。
- 私有代码库应明确哪些目录可纳入上下文，避免泄露敏感信息。
